# Рядом

MVP русскоязычной соцсети по состоянию: пользователь пишет короткое состояние и мысль, выбирает намерение, проходит safety-фильтр, получает подбор похожих людей и попадает в малую анонимную комнату.

## Что реализовано

- Mobile-first PWA на React + Vite + TypeScript.
- Чек-ин: короткое состояние и мысль, четыре намерения.
- Локальный алгоритм матчинга: темы, совместимость намерения, свежесть, trust-score.
- Safety-фильтр: кризисные и запрещенные контексты не отправляются в обычный матчинг.
- Комната на 2-5 участников с таймером, realtime-сообщениями и подсказками для начала разговора.
- Совместный цикл «Искра → Зеркало → След»: Зеркало открывается только после реплик двух участников.
- Приватный Атлас: выбранный След сохраняется локально без полной истории переписки.
- Репорт и блокировка через компактное меню с записью в очередь модерации.
- Добровольная оценка разговора без автоматической подстановки результата.
- Production-слой на Supabase: anonymous auth, Postgres-таблицы, RLS, RPC для создания комнаты, realtime-сообщения, waitlist, reports, feedback.
- Demo fallback: если Supabase env не задан, приложение остается кликабельным локальным MVP.
- PWA-манифест и service worker для базового офлайн-кэша.

## Live backend через Supabase

1. Создай Supabase project.
2. Примени все SQL-файлы из `supabase/migrations` по имени, от старого к новому.
3. Для anonymous-auth режима включи Anonymous Sign-Ins и добавь `VITE_SUPABASE_USE_AUTH=true`. Без этого флага приложение использует защищенную гостевую RPC-сессию без регистрации.
4. Создай `.env.local` по примеру `.env.example`.
5. Запусти `npm run dev`.

```bash
VITE_SUPABASE_URL=https://your-project-ref.supabase.co
VITE_SUPABASE_PUBLISHABLE_KEY=sb_publishable_your_key_here
```

Фронтенд не использует service role key. Пользовательские операции проходят через RLS и ограниченные RPC. `smoke:live` создает два временных тестовых чек-ина и проверяет одну общую комнату, обновление участников и доставку сообщения.

Подробная инструкция: `docs/production-setup.md`.
